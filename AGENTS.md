# Project Rules & Agent Instructions

## App Goals
- **Application Scope:** An Instagram-like social media application built with Jetpack Compose.
- **Customizable Navigation:** The bottom navigation bar must support toggling specific tabs (Home, Search, Add, Snaply, Account) through user preferences located in the Settings screen.
- **Terminology:** The standard "Story" feature must always be referred to as **"Snaply"**.

## Security & Authentication (Critical)
- **Data Privacy:** User data must never be leaked or exposed.
- **Robust Auth Flow:** Login and sign-up logic must not break. It must be implemented securely (using recommended local or remote authentication patterns).
- **Authorization:** Only strictly authorized/logged-in users are permitted to access their personal feeds, settings, or private data. 

## Communication & Coding Style
- **Code Explanation:** When building or modifying features, you MUST explain every piece of code you write in a clear, easy-to-understand manner. Break down the logic so the user can follow along and learn.
